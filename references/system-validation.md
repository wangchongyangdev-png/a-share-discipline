# 系统验证与交易样本

这份规则用于把交易系统从“感觉有效”变成“可复盘、可修正”。

## 核心原则

一条规则不能靠一笔交易验证。

至少用：

- `20` 笔交易看执行问题
- `50` 笔交易看初步胜率
- `100` 笔交易看系统稳定性

不要因为一笔亏损就否定规则，也不要因为一笔盈利就放大仓位。

## 每笔交易必须记录

字段：

- 日期
- 标的
- 类型：ETF / 个股 / 宽基
- 板块地位：主线 / 次主线 / 轮动 / 弱势 / 宽基不适配
- 趋势阶段：启动 / 主升 / 高位震荡 / 调整 / 退潮
- 触发规则：MA5 回踩 / 站回 MA10 / 回踩 MA10 / 横盘突破 / 其他
- 买入仓位
- 买入是否符合规则
- 卖出触发条件
- 盈亏
- 是否情绪交易
- 复盘结论

## 统计指标

每 `20` 笔统计一次：

- 胜率
- 平均盈利
- 平均亏损
- 最大单笔亏损
- 最大回撤
- 违反规则次数
- 最常见情绪错误

判断标准：

- 胜率低但盈亏比高：不一定坏
- 胜率高但亏损单很大：危险
- 违反规则次数多：先修执行，不急着改规则

## 规则修改原则

先修执行，再修规则。

只有在满足下面条件时，才修改规则：

- 至少有 `20` 笔同类样本
- 不是因为最近一笔亏损而临时改
- 修改后能写成更清晰的触发条件

不要做：

- 今天亏了，就把 MA10 改成 MA20
- 今天卖飞了，就取消止盈
- 今天没买到，就允许明天无条件追

系统要越来越清晰，而不是越来越随意。
