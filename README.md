# A 股趋势交易纪律 Skill

一个面向 Codex 的个人交易纪律助手，用于判断 A 股 ETF 为主、A 股短线趋势交易为辅的标的，是否符合一套可重复执行的短线趋势系统。

它不预测“明天一定涨不涨”，也不输出“强烈买入”。它更关注：这是不是系统内交易、方向处于什么阶段、当前位置是否舒服，以及此刻最该做什么。

## 它解决什么问题

很多交易判断的问题不在于不会找热点，而在于：

- 热门方向已经很强，但买点可能很差
- 一上涨就想追，一回调又不敢买
- 赚了不会卖，亏了又容易死扛
- 盘中波动带来情绪化交易
- 只看均线或只看消息，缺少完整的执行框架

这个 Skill 会先核验代码、价格和需要时的消息证据，再把结论翻译成交易纪律语言：

- 这是买点、持有点、减仓点，还是观察点？
- 它是主线、次主线、轮动，还是弱势方向？
- 现在处于启动、主升、调整还是退潮？
- 趋势很强，但位置是否已经不适合买？
- 此刻最容易犯的情绪错误是什么？

## 核心原则

- 强，不等于可以买
- 便宜，不等于安全
- 只做高胜率位置，不做模糊位置
- 不吃头，不吃尾，只吃中间
- 不预测，只应对
- 买点决定胜率，卖点决定盈亏比，仓位决定账户波动
- 仓位乱了先修仓位，不进攻
- 卖飞不是错误，盈利回吐成亏损才是错误

## 适用范围

- 主要：任意 A 股 ETF，包括行业、主题、商品、港股相关和宽基 ETF
- 次要：明显适合同一套短线趋势纪律的 A 股个股

它不会把半导体、创新药、券商等写死为固定主线；每次分析都会先判断标的当前在市场中的地位和阶段。

## 安装

本仓库采用“仓库根目录 + 同名 Skill 子目录”的结构：

```text
仓库根目录/
├── README.md
└── a-share-trend-discipline/
    ├── SKILL.md
    ├── agents/
    └── references/
```

发布到 GitHub 后，用户可以让 Codex 安装该 GitHub 路径，或在具备 Skill Installer 的环境中运行：

```bash
python ~/.codex/skills/.system/skill-installer/scripts/install-skill-from-github.py \
  --repo <你的 GitHub 用户名>/a-share-discipline \
  --path a-share-trend-discipline
```

安装完成后，Skill 会在下一轮对话中可用。使用示例：

```text
$a-share-trend-discipline 516510 现在是买点还是观察点？
```

## 数据与风险提示

此 Skill 是交易纪律和结构判断工具，不构成投资建议，也不能保证收益。

当用户问“今天”“最新”“消息驱动”等实时问题时，它会把代码、价格、消息拆开核验：证据不足时会明确降级为结构判断，不把截图或历史页面伪装成实时数据。
